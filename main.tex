\begin{frame}[plain]
\contentheader{Executive Takeaway}
\begin{columns}[T,totalwidth=\textwidth]
\begin{column}{0.52\textwidth}
\begin{block}{一句话结论}
LOB 研究的价值不只是预测下一跳价格，而是把市场状态拆成 \key{方向、流动性、可成交性、风险约束}，判断信号能否转化为真实 P\&L。
\end{block}
\begin{itemize}\tightitems
  \item 短周期价格预测容易被 spread、impact、queue position 吃掉。
  \item 订单簿状态提供比价格序列更接近交易执行的状态变量。
  \item 内网数据应重点验证：信号方向是否稳定、流动性是否足够、执行成本是否可控。
\end{itemize}

\begin{frame}[plain]
\contentheader{What a LOB State Contains}
\begin{columns}[T,totalwidth=\textwidth]
\begin{column}{0.50\textwidth}
\begin{itemize}\tightitems
  \item 每个价位有 price 和 queue size：\(bid_i, ask_i, q^b_i, q^a_i\)。
  \item 常用输入是最近 \(T\) 个时刻、前 \(L\) 档订单簿：
  \[
  X_t \in \mathbb{R}^{T\times 4L}.
  \]
  \item 关键不是一根价格线，而是 bid/ask 两侧队列随时间的形状变化。
\end{itemize}
\begin{block}{近端不平衡}
\[
I_t^{(k)}=\frac{\sum_{i=1}^k q^b_i-\sum_{i=1}^k q^a_i}{\sum_{i=1}^k q^b_i+\sum_{i=1}^k q^a_i}
\]
\end{block}
\end{column}

\begin{frame}[plain]
\contentheader{Why Liquidity Is the True State Variable}
\begin{columns}[T,totalwidth=\textwidth]
\begin{column}{0.52\textwidth}
\begin{itemize}\tightitems
  \item Direction tells us where price may move.
  \item Liquidity tells us whether the move can be monetized.
  \item Execution converts a prediction into realized P\&L.
\end{itemize}
\begin{block}{Decision state}
\[
s_t=(price,\ book,\ liquidity,\ execution,\ risk).
\]
\end{block}

\begin{frame}[plain]
\contentheader{Modeling Pattern: DeepLOB as a Useful Baseline}
\begin{columns}[T,totalwidth=\textwidth]
\begin{column}{0.48\textwidth}
\begin{itemize}\tightitems
  \item CNN learns local price-level and bid/ask interaction patterns.
  \item Inception-style layers capture different local scales.
  \item LSTM/sequence block captures recent temporal dynamics.
  \item Output is usually a short-horizon up/down/stationary label.
\end{itemize}
\begin{block}{Practical message}
Deep model matters less than having correct labels, leak-free evaluation, and executable cost-aware metrics.
\end{block}
\end{column}

\begin{frame}[plain]
\contentheader{From Prediction to Trading Signal}
\begin{columns}[T,totalwidth=\textwidth]
\begin{column}{0.51\textwidth}
\begin{block}{A signal is tradable only if}
\[
E[\Delta p] > spread + fee + impact + delay\ risk.
\]
\end{block}
\begin{itemize}\tightitems
  \item Direction: expected sign and confidence.
  \item Liquidity: depth, spread, imbalance, resiliency.
  \item Execution: aggressive/passive choice, queue priority, slippage.
  \item Risk: holding horizon, inventory, adverse selection.
\end{itemize}
\end{column}

\begin{frame}[plain]
\contentheader{Recommended Research Workflow}
\begin{columns}[T,totalwidth=\textwidth]
\begin{column}{0.50\textwidth}
\begin{enumerate}
  \item Build clean LOB snapshots and event-aligned labels.
  \item Start with interpretable features: spread, depth, imbalance, impact proxy.
  \item Benchmark simple models before DeepLOB.
  \item Add execution filter and cost-aware backtest.
  \item Only then test deep sequence models.
\end{enumerate}
\end{column}
\begin{column}{0.46\textwidth}
\begin{block}{Minimum viable internal study}
\begin{itemize}\tightitems
  \item 3--5 liquid instruments.
  \item Multiple market regimes: open, normal, volatile, close.
  \item Report: signal IC, turnover, cost, realized slippage, capacity.
\end{itemize}
\end{block}
\end{column}
